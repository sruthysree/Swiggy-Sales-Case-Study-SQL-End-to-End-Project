# Swiggy-Sales-Case-Study-SQL-End-to-End-Project
This project demonstrates an end-to-end SQL case study analyzing Swiggy’s sales data. It covers data validation, cleaning, schema design (star schema with dimension and fact tables), and advanced reporting. The workflow transforms raw transactional data into structured insights for business decision-making.

Objectives

Validate and clean data (nulls, blanks, duplicates).

Build dimensional schema (dim_date, dim_location, dim_restaurant, dim_category, dim_dish).

Create fact table for orders and KPIs.

Analyze trends: monthly, quarterly, yearly, and by day of week.

Identify top cities, restaurants, categories, dishes, and revenue contributions.

Provide actionable insights on pricing, ratings, and cuisine performance.

Key Insights

Top 10 cities contribute the majority of Swiggy’s order volume.

Restaurants with higher ratings show stronger revenue growth.

Price range 100–199 INR dominates overall sales, highlighting customer affordability preferences.

Most ordered dishes reveal popular cuisines driving repeat orders.

State-level revenue analysis shows regional hotspots for expansion.

Category performance indicates which cuisines combine high order volume with strong average ratings.
